# API Reference — الحالة الفعلية بعد المطابقة مع الباك إند الحقيقي

هذا الملف يوثّق كيف تستهلك الواجهة الأمامية (لجنة الإشراف بشكل أساسي، وأجزاء مشتركة من المصادقة والصفحة العامة) الباك إند **الحقيقي**، بعد مطابقته سطرًا بسطر مع `docs/api-reference.html` (الدليل الكامل المرسل من فريق الباك إند — احتفظي به بالمجلد، لا حاجة لإعادة رفعه).

⚠️ **لا توجد بيانات وهمية في أي مكان بالكود بعد الآن.** كل شاشة إما تستهلك endpoint حقيقي موثّق، أو تعرض رسالة فجوة صريحة (بطاقة صفراء أعلى الصفحة) بدل تحميل بلا نهاية أو أرقام مصطنعة.

---

## المصادقة (`auth.api.js` + `auth.store.js`)

| نداء | المسار الحقيقي | ملاحظة |
|---|---|---|
| تسجيل الدخول | `POST /login` | `{ email, password }` → `{ user, token }` |
| تسجيل الخروج | `POST /logout` | |
| المستخدم الحالي | `GET /me` | لاستعادة الجلسة عند إعادة التحميل |
| تغيير كلمة المرور | `POST /me/change-password` | معفى من بوابة 423 |
| قبول دعوة (أول دخول) | `POST /invite/{token}/accept` | `{ password, password_confirmation }` — تعيد استخدامها `ResetPasswordPage.vue` |

**فجوة:** لا يوجد "نسيت كلمة المرور" ذاتي الخدمة بهذا الباك إند إطلاقًا — كلمات المرور تُدار حصرًا عبر روابط دعوة لمرة واحدة (3 أيام). `ForgotPasswordPage.vue` ما زالت موجودة بالكود لكن `forgotPassword()` بالـ store ترفض دومًا برسالة صريحة بدل مناداة endpoint غير موجود. **قرار منتج مطلوب:** إخفاء رابط "نسيت كلمة المرور" من `LoginPage.vue` بالكامل، أو إبقاؤه موجّهًا لتواصل يدوي مع لجنة الإشراف.

**بوابة 423:** أي طلب (عدا الاثنين أعلاه) لمستخدم `must_change_password: true` يرجع HTTP 423. `api.js` ينسّق هذا كـ `error.normalized.mustChangePassword = true`، لكن **لا توجد بعد شاشة/توجيه فعلي يستهلك هذا العلم** — يحتاج بناء شاشة "غيّري كلمة المرور أولًا" وربطها براوتر-جارد.

## لوحة لجنة الإشراف (`committee.api.js` + `committee.store.js`)

الباك إند **غير مقسّم بمسارات لكل دور** — كل الموارد عامة الشكل (`/teams`, `/discussions`, `/proposals`)، والصلاحية تُحسم بدور المستخدم المسجّل دخوله لا بمسار خاص.

| المورد | المسار | تُستخدم في |
|---|---|---|
| كل الفرق | `GET /teams` | Dashboard, Teams, Members, Archive, Progress (كلها مشتقّة من نفس الاستدعاء) |
| فريق واحد | `GET /teams/{id}` | غير مستخدم حاليًا (تكفي القائمة) |
| إنشاء فريق | `POST /teams` | TeamsPage — `{name, supervisor_id, specialization_id, member_ids, leader_id}` |
| معاينة استيراد طلاب | `POST /teams/import/preview` | TeamsPage — `FormData{file}` |
| تأكيد استيراد طلاب | `POST /teams/import/confirm` | TeamsPage — `{specialization_id, rows}` |
| تقدّم فريق واحد | `GET /teams/{id}/progress` | ProgressPage — يُستدعى لكل فريق على حدة (لا نسخة جماعية) |
| تصدير التقدّم | `GET /progress/export` | ProgressPage — ملف xlsx مباشر |
| مواعيد المناقشات | `GET /discussions` | AppointmentsPage |
| تصدير المواعيد | `GET /discussions/export` | AppointmentsPage — ملف xlsx مباشر |
| إنشاء موعد مناقشة | `POST /discussions` | AppointmentsPage — `{project_id, supervisor_id, place, discussion_date, discussion_time, committee, whatsapp, status}` |
| موافقة على مقترح | `POST /proposals/{id}/approve` | ProposalsPage |
| رفض مقترح | `POST /proposals/{id}/reject` | ProposalsPage — `{rejection_reason}` |
| إعادة إرسال دعوة | `POST /users/{id}/invite` | MembersPage — لعضو **موجود سلفًا** فقط |
| الأقسام / التخصصات | `GET /departments`, `GET /specializations` | فلاتر (تتطلب تسجيل دخول، متاحة هنا) |

### فجوات حقيقية أثّرت على تصميم كل صفحة

1. **لا توجد لوحة تحكم مجمّعة.** `DashboardPage.vue` تحسب كل شي محليًا من `GET /teams` (عدد الفرق، الطلاب، المشرفين، توزيع حالات المشاريع). "متوسط الإنجاز" **غير محسوب هنا** لأنه يتطلب استدعاء منفصل لكل فريق (بطيء) — الصفحة تعرض بطاقة توضيحية توجّه لصفحة "نسبة تقدّم المشاريع" بدلًا من رقم مصطنع.
2. **لا إدارة مستخدمين عامة.** لا يوجد endpoint لإنشاء مشرف أو عضو لجنة إشراف جديد — فقط استيراد طلاب بالجملة. `TeamsPage.vue` تعرض بطاقة تحذير بذلك، وقائمة "المشرفين" بكل مكان مشتقّة فقط ممن لديهم فريق بالفعل (لا طريقة لإضافة مشرف جديد لا فريق له بعد).
3. **لا تعديل/حذف فريق أو عضو بعد إنشائه.** `TeamsPage.vue` تعرض فقط: إنشاء فريق جديد + استيراد طلاب + عرض القوائم. لا أزرار تعديل/حذف (كانت موجودة بنسخة سابقة مبنية على افتراض خاطئ لعقد الـ API، أُزيلت بالكامل).
4. **`MembersPage.vue` بلا endpoint مستقل.** الطلاب والمشرفون مُشتقّون من `GET /teams` (getters بالـ store: `studentsFromTeams` / `supervisorsFromTeams`). عمود كلمة السر أُزيل بالكامل (الباك إند لا يرسلها أبدًا). زر "تقييد الدخول" استُبدل بـ "إعادة إرسال رابط دعوة" (نداء حقيقي).
5. **`ProposalsPage.vue` بلا قائمة.** الموافقة/الرفض حقيقيان 100%، لكن لا يوجد أي endpoint موثّق لعرض قائمة المقترحات المعلّقة. الصفحة تكتفي بإجراء يدوي عبر إدخال رقم المقترح. **هذه أهم فجوة تحتاج توضيح من فريق الباك إند** — إما endpoint قائمة فعلي، أو مصدر بديل (مثلًا حقل `proposal` مضمّن داخل رد `GET /teams`).
6. **`AppointmentsPage.vue` (مواعيد المناقشات) نموذج بيانات مختلف جذريًا** عمّا افترضته النسخة السابقة: سجل واحد لكل **مشروع** (لا مجموعة طلاب)، رقم واتساب واحد فقط، ولا تعديل/حذف بعد الإنشاء. النموذج الجديد يختار "الفريق" من القائمة المعروفة ويشتق منه `project_id` و`supervisor_id` تلقائيًا.
7. **`ProjectArchivePage.vue` بلا مفهوم "أرشيف" حقيقي.** لا حالة "منتهي" تتفعّل تلقائيًا، ولا endpoint لتمييز مشروع بنجمة أو لعرض ملفاته. الصفحة تعرض كل المشاريع المرتبطة بفريق (من `GET /teams`) بصرف النظر عن حالتها. **`ProjectDetailPage.vue` حُذفت بالكامل** لعدم وجود أي endpoint لتفاصيل مشروع مفرد.
8. **`ProgressPage.vue` تستدعي `GET /teams/{id}/progress` لكل فريق بالتوازي** (`Promise.allSettled`) عند تحميل الصفحة — قد تلاحظين تأخرًا مع كثرة الفرق، هذا محدود ببنية الباك إند لا الفرونت. التصدير الجماعي (`GET /progress/export`) حقيقي وسريع.

## الصفحة العامة (`public.api.js` + `landing.store.js`)

المسار العام الحقيقي الوحيد: **`GET /projects/featured`** (أفضل 6 مشاريع، بدون مصادقة).

**لا يوجد بهذا الباك إند:** إحصائيات عامة (`fetchStats`)، تصفح كل المشاريع بترقيم صفحات وفلاتر (`fetchProjects` بصفحة الأرشيف العامة)، ولا أقسام عامة بدون تسجيل دخول (`fetchDepartments` — المسار الحقيقي `/departments` موجود لكنه محمي). الثلاثة أُبقيت كدوال بالـ store لتفادي كسر المكوّنات الحالية، لكنها تضبط حالة خطأ واضحة فورًا **بدون مناداة أي endpoint وهمي**، والمكوّنات (`StatsBanner`, `DepartmentsSection`, `ProjectsArchivePage`) تعرض هذه الرسالة بدل بيانات مصطنعة.

## فجوات عامة عبر التطبيق (خارج نطاق الصفحات المبنية حاليًا)

هذه فجوات حقيقية موثّقة بـ `docs/api-reference.html` تمس بنية مشتركة (Layout/Topbar) تُستخدم بكل الأدوار، لم تُعدَّل بهذه الجولة لأنها تمس نطاقًا أوسع من صفحات لجنة الإشراف:

- **الفصل الدراسي (Semester Selector):** لا يوجد `GET /semesters` أو ما يعادله لتعبئة القائمة المنسدلة بـ `AppTopbar.vue` — الباك إند يفترض الفصل الحالي تلقائيًا إن لم يُرسل `term_id`. القائمة حاليًا ستبقى فارغة/معطّلة بصمت.
- **الإشعارات (`NotificationBell.vue`):** جدول `notifications` موجود بقاعدة البيانات، لكن صفر endpoints. الجرس سيبقى بلا بيانات.
- **الملف الشخصي العام (`shared.api.js` → `/profile`):** لا مسار مطابق موثّق؛ البديل الحقيقي `GET /me` (قراءة فقط، لا تعديل موثّق).
- **الأدوار الأربعة الأخرى (سوبر أدمن، مشرف، قائد فريق، طالب):** ملفات `*.api.js` الخاصة بها ما زالت فارغة تمامًا — لا صفحات مبنية لها بعد (فقط `PlaceholderPage`). محتوى `docs/api-reference.html` أقسام 4، 6-8 يغطّي كل ما تحتاجه هذه الأدوار وقت بنائها.

راجعي `docs/api-reference.html` للتفاصيل الكاملة بكل حقل، رسائل الخطأ، وحدود معدّل الطلبات.
